Section 3 — Roadmap d'engagement

Recommandations priorisées par impact sur la fidélisation · Phase A→B

🚀 Court terme — Impact immédiat, effort faible
1

Renommer l'onglet "Invitations" → "Activité" effort: bas

Regrouper dans un vrai feed : demandes d'amis, sorties reçues, + nouveaux spots ajoutés par tes amis, nouveaux likes sur tes spots. C'est le principal driver de retour quotidien sur une app sociale. Instagram Stories → notifications → retour. Même logique.

2

Remonter "Créer une sortie" depuis le panel d'un spot effort: bas

Ajouter un bouton "📅 Proposer ici" dans le panel spot (à côté de Navigation et Partage). C'est le flow le plus naturel : tu vois un spot cool → tu invites un ami à y aller. Actuellement cette feature est introuvable.

3

Remonter le bouton "J'ai visité" dans le panel spot effort: bas

Actuellement sous le fold (après scroll). Le monter juste sous le titre/like, à côté du ❤️. C'est une action sociale forte — plus visible = plus de visites enregistrées = plus d'engagement dans le cercle d'amis.

4

Badge "Nouveau" sur les pins de spots récents effort: bas

Un petit dot ou halo sur les spots ajoutés < 48h. Donne une raison de rouvrir l'app : "y'a des nouveaux spots depuis hier ?" La logique isNew() existe déjà dans ExploreModal — juste à l'utiliser sur la carte.

5

Afficher le classement dans le profil effort: bas

Sur la page Profil, afficher "Tu es #2 ce mois · 5 spots ajoutés". La compétition amicale est un driver de retour fort. Le classement existe, il est juste dans le mauvais endroit (FriendsModal > onglet 2 uniquement).

⚙️ Moyen terme — Engagement + croissance A→B
6

Notifications push PWA effort: moyen

Sans push, l'app ne peut pas ramener les utilisateurs passifs. Priorité : notif quand un ami ajoute un spot, quand on est invité à une sortie, quand quelqu'un like ton spot. Le Service Worker est déjà impliqué via Next.js — c'est faisable.

7

Tips / micro-commentaires sur les spots effort: moyen

Pas des commentaires complets — juste des tips courts (max 120 chars). "Prends la table du fond", "Évite le week-end, trop bondé". Crée de la valeur sociale et donne envie de revenir voir ce que les amis ont dit.

8

Partage de profil public effort: moyen

Une page /u/username avec les spots publics et stats. Le partage de spot existe déjà (/spot/[id]) — même pattern. Permet à un ami de découvrir l'app via le profil d'un ami (vecteur de croissance A→B).

9

Réorganiser FriendsModal : onglet "Sorties" dédié effort: moyen

Sortir les Sorties de l'onglet "Invitations" vers leur propre onglet ou section dédiée. Ou mieux : ajouter un 4e bouton dans la bottom nav "Sorties 🗓️" avec badge pour les invitations en attente. Cette feature mérite sa propre entrée.

🔮 Long terme — Phase B/C, croissance virale
10

Spots éphémères mis en avant effort: élevé

La colonne expires_at est déjà en DB. Créer une UI type "Stories" en haut de l'Explorer pour les spots éphémères actifs (concert ce soir, marché du week-end). Urgence = engagement fort.

11

Mode découverte géo-étendu effort: élevé

En phase B/C, permettre de voir les spots de personnes qu'on ne suit pas encore (amis d'amis, public dans un rayon). Le mode "Découverte" est déjà géré pour les users sans amis — il suffit de l'étendre avec des contrôles de confidentialité clairs.

12

Recommandations intelligentes effort: élevé

"Tes amis adorent les cafés dans le 11e — voici 3 spots qu'ils ont likés". Utilise les données existantes (likes, catégories, géo) pour personnaliser Explorer. Forte rétention, virale si les reco sont bonnes.

Pour passer de A à B : les 5 priorités absolues sont :

1. Feed d'activité (savoir ce que font les amis)

2. Sorties accessibles depuis le panel spot

3. Notifications push PWA

4. Tips courts sur les spots

5. Partage de profil public
